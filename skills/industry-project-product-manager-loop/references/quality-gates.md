# 项目质量门

## 使用方法

质量门用于判断项目是否真正具备进入下一阶段的条件。每项标记：

- `PASS`：证据和内容充分；
- `PARTIAL`：可继续但存在明确缺口；
- `FAIL`：不能进入下一阶段；
- `N/A`：经说明后不适用。

## Gate A：机会有效

必须确认：

- 问题发生在明确的角色和场景中；
- 问题频率、影响或风险至少有一种证据；
- 当前替代方案及不足已识别；
- 为什么现在值得做有合理解释；
- 产品机会不是从已有功能反推出来的；
- 关键假设已登记。

阻断条件：

- 只有宏观趋势，没有具体客户问题；
- 痛点来自想象，没有验证计划；
- 项目目标仅为“建设一个平台”。

## Gate B：客户、买方与用户清晰

必须确认：

- 目标客户细分明确；
- 购买者、决策者、使用者、管理员、影响者已区分；
- 关键角色的任务、流程和指标已定义；
- 至少一个核心用户旅程已还原；
- 购买触发、预算来源和采购障碍有初步判断；
- 用户需求与客户商业需求未混为一谈。

阻断条件：

- Persona只有年龄、性别等人口属性；
- 目标用户写成“所有企业”或“运维人员”；
- 没有真实场景或任务。

## Gate C：策略和差异化成立

必须确认：

- 竞品和替代方案选择有理由；
- 比较维度与客户购买决策相关；
- 清楚说明产品不做什么；
- 已形成Build、Buy、Integrate或Partner判断；
- 差异化可以落实为能力、流程、数据或交付优势；
- 商业模式与目标客户采购方式基本匹配。

阻断条件：

- 竞品分析只统计功能数量；
- 差异化只写“更智能、更易用”；
- 产品边界无限扩张。

## Gate D：项目可立项

必须确认：

- 目标和成功指标可衡量；
- In Scope和Out of Scope明确；
- 主要交付物明确；
- 项目负责人和关键干系人明确；
- 时间、预算、人员和技术约束已记录；
- 风险和依赖有负责人；
- Roadmap按阶段目标而不是功能堆叠；
- P0需求有优先级依据。

阻断条件：

- 成功标准只有“按时上线”；
- 没有决策人；
- 范围与资源明显不匹配；
- 关键外部系统和数据依赖未确认。

## Gate E：需求可实现、可测试、可追溯

必须确认：

- 每个P0需求能追溯到上游需求或风险；
- 主流程、分支流程和异常流程完整；
- 状态与状态转换明确；
- 角色、权限和数据范围明确；
- 业务规则无明显冲突；
- 字段和数据来源可定义；
- 非功能要求量化；
- 验收标准可以被测试；
- 埋点和指标与产品目标对应；
- 原型、PRD和流程图保持一致。

阻断条件：

- 需求只能通过“与原型一致”验收；
- P0需求没有异常处理；
- 高风险动作没有权限、审批或审计；
- 指标没有统计口径。

## Gate F：工程和交付可行

必须确认：

- 系统边界和责任边界清楚；
- 外部接口清单和负责人明确；
- 关键技术风险已评估；
- 部署方式和环境约束明确；
- 安全、合规和审计要求已进入设计；
- 性能、容量、可用性和容灾目标明确；
- 升级、兼容和回滚方案可行；
- 数据迁移和历史数据处理有计划；
- 测试环境和测试数据可获得；
- 运维责任和支持流程明确。

阻断条件：

- 依赖系统未确认接口；
- 私有化项目没有部署和升级方案；
- 安全要求准备在上线前再处理；
- 数据迁移不可逆且没有验证步骤。

## Gate G：上线准备完成

必须确认：

- P0测试通过；
- UAT完成或有正式豁免；
- 已知问题和影响明确；
- 上线步骤、负责人和时间窗口明确；
- 灰度或分批策略明确；
- 核心监控和告警已配置；
- 回滚条件、步骤和负责人明确；
- 数据备份或恢复方案验证；
- 培训、手册和支持渠道就绪；
- 上线后的观察窗口和成功判定明确。

阻断条件：

- 只能上线，不能回滚；
- 没有业务和技术值班人；
- 关键指标上线后无法观测；
- 客户不知道如何获取支持。

## Gate H：价值验证和复盘完成

必须确认：

- 上线结果与原目标使用同一口径比较；
- 功能使用和核心任务完成情况可分析；
- 业务价值、成本和稳定性变化有数据；
- 用户反馈和问题已归类；
- 重大故障或延期有根因分析；
- 未达目标的原因已区分产品、交付、运营和市场因素；
- 下一版本决策有证据；
- 可复用经验已回灌到模板、知识库或Skill。

阻断条件：

- 只汇报上线功能数量；
- 用目标值冒充实际结果；
- 没有记录失败和取舍；
- Roadmap未根据结果更新。

## 完成度评分

建议每个Gate按以下方式计算：

- PASS = 2分；
- PARTIAL = 1分；
- FAIL = 0分；
- N/A不计入分母。

阶段完成率：

```text
实际得分 ÷ 可获得总分 × 100%
```

分数只用于发现缺口，不能替代关键阻断项判断。任一阻断项存在时，即使总分较高也不得判定通过。

## 最终退出标准

大型ToB项目至少满足：

- Gate A—E全部PASS；
- Gate F和G无FAIL；
- 上线前不要求Gate H完成，但必须有Gate H的测量计划；
- 所有P0需求有验收标准；
- 所有高风险事项有负责人和处置方案；
- 所有关键数字有来源或明确标注为目标/假设。
