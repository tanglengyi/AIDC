# Skill行为测试场景

这些场景用于人工或Agent测试，检查Skill是否按预期工作。

## 场景1：陌生行业从0开始

### 输入

```text
行业：工业机器人
项目：面向中型制造企业的机器人远程运维平台
目标：大型ToB解决方案
现有材料：无
```

### 预期行为

- 建立Project Brief和状态文件；
- 明确未知信息和研究计划；
- 不直接生成看似完整的PRD；
- 先研究产业链、客户类型、购买委员会和真实运维流程；
- 将市场规模、客户痛点和收益数字标记为待验证；
- 给出下一轮最高价值动作。

### 失败表现

- 编造客户访谈；
- 未界定项目范围就生成全部页面；
- 将所有制造企业视为同一客户；
- 没有停止条件。

## 场景2：已有PRD但缺上游论证

### 输入

```text
检查当前目录。已经有一份5000字PRD，但没有市场、用户和竞品材料。
```

### 预期行为

- 识别PRD存在；
- 不把项目判定为完整；
- 检查P0需求是否能追溯到痛点和用户任务；
- 生成上游证据缺口；
- 优先补充机会分析、用户研究和竞争策略；
- 标记现有PRD中没有依据的功能。

## 场景3：竞品分析只有功能勾选

### 输入

```text
竞品表有十个产品和五十项功能，请检查是否足以指导产品规划。
```

### 预期行为

- 检查竞品选择逻辑、客户、部署、商业模式和实施边界；
- 判断功能矩阵只是输入，不是结论；
- 输出Build/Buy/Integrate建议；
- 明确应该做什么、不做什么以及原因；
- 不以功能数量判断竞品优劣。

## 场景4：客户要求“全部自动修复”

### 输入

```text
客户希望智能运维平台发现故障后全部自动修复，不需要人工审批。
```

### 预期行为

- 识别生产高风险；
- 建立动作风险分级；
- 低风险动作可自动执行；
- 高风险动作要求审批、影响预览、回滚和审计；
- 将最终风险接受升级给客户或项目负责人；
- 不直接承诺全自动。

## 场景5：材料齐全但数字无来源

### 输入

```text
目录中BP、BRD、MRD、PRD和上线方案齐全，但都写着“降低30%成本、提升50%效率”。
```

### 预期行为

- 结构审计可显示文件存在；
- 语义质量门必须标记证据不足；
- 区分客户基线、行业参考、产品目标和实际结果；
- 建立验证方法和统计口径；
- 不将项目判定为已验证。

## 场景6：项目范围膨胀

### 输入

```text
一期原本只做GPU资产纳管，现在客户又要求DCIM、Slurm、MLOps、MaaS、计费和数字孪生全部上线。
```

### 预期行为

- 回到PID和范围管理；
- 分析业务闭环、依赖、资源和时间；
- 提供多个阶段方案；
- 明确一期In Scope和Out of Scope；
- 将重大范围变化升级决策；
- 不静默扩展PRD。

## 场景7：上线前检查

### 输入

```text
研发说功能已经开发完成，明天上线。
```

### 预期行为

检查：

- P0测试；
- UAT；
- 已知问题；
- 数据迁移；
- 灰度策略；
- 监控告警；
- 回滚条件；
- 值班和支持；
- 培训手册；
- 上线观察窗口。

任一关键阻断项失败时，不应仅因开发完成而判定可上线。

## 场景8：面试作品集

### 输入

```text
把当前AIDC项目整理成产品经理面试作品集。
```

### 预期行为

- 选择Interview Portfolio档位；
- 用“业务问题→用户证据→产品决策→交付→结果→复盘”组织；
- 选取一到两个PRD深挖，而不是展示全部文档；
- 区分真实成果与目标指标；
- 突出取舍、优先级和跨团队协作；
- 输出还缺哪些真实项目数据。

## 场景9：Loop无新增证据

### 输入

```text
连续三轮都在扩写同一份市场分析，没有新数据、访谈或来源。
```

### 预期行为

- 停止机械扩写；
- 将状态设为`evidence_exhausted`；
- 输出调研、访谈或数据获取计划；
- 不用更多形容词制造虚假进展。

## 场景10：跨文档冲突

### 输入

```text
BRD写一期只做私有化，PRD支持SaaS，部署方案又只支持单机安装。
```

### 预期行为

- 识别BRD、PRD和部署方案冲突；
- 判断冲突影响范围；
- 找出对应决策人；
- 给出选项和建议；
- 冲突解决前Gate D、E或F不得通过。

## 场景11：审计脚本误判

### 输入

```text
有一个名为PRD.md的文件，内容只有“待补充”。
```

### 预期行为

- 脚本将其标记为`thin_content`或`partial`；
- Agent不能仅凭文件名判定PRD完成；
- 输出需补充的结构和证据。

## 场景12：项目已经成功完成

### 输入

```text
所有必选文档已通过评审，P0测试与UAT通过，上线和回滚就绪，关键指标可以观测。
```

### 预期行为

- 上线前可将Loop状态置为`complete`或进入运营观察阶段；
- Gate H保持为待验证，而不是伪造上线结果；
- 明确上线后复盘时间和统计口径；
- 不继续创建无价值的文档。
