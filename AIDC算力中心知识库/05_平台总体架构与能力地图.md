# AIDC运营管理平台总体架构与能力地图

## 1. 平台定位

AIDC运营管理平台定位为面向智算中心的综合运营与智能运维平台。它连接数据中心设施、GPU服务器、高速网络、存储、Kubernetes/Slurm调度系统、AI开发平台、模型推理服务和客户运营体系，实现算力资源从建设、纳管、调度、使用、计量、计费到经营分析的闭环。

平台不是单纯DCIM，也不是单纯Kubernetes平台或MLOps平台，而是跨层集成与运营中枢。

## 5. 关键业务闭环

### 5.1 资源交付闭环

```text
资源入库 → 资产上架 → 健康检测 → 加入资源池 → 产品上架 → 客户申请 → 审批 → 自动开通 → 使用监控 → 到期回收
```

### 5.2 故障处理闭环

```text
监控发现 → 告警聚合 → 根因分析 → 影响面判断 → 通知值班 → 创建工单 → 执行Runbook → 恢复验证 → 复盘报告
```

### 5.3 计量计费闭环

```text
产品定价 → 订单开通 → 用量采集 → 计费计算 → 账单生成 → 客户对账 → 开票收款 → 续费/退订
```

### 5.4 容量规划闭环

```text
容量采集 → 利用率分析 → 增长预测 → 瓶颈识别 → 采购建议 → 扩容实施 → 资源入池 → 效果评估
```

## 7. 建设阶段建议

### 阶段一：可视化与纳管

目标：看得见、管得住。

建设内容：资产、机房、GPU服务器、基础监控、告警工单、容量能耗。

### 阶段二：资源池与调度

目标：调得动、用得好。

建设内容：Kubernetes/Slurm集成、GPU资源池、队列、配额、作业管理、利用率分析。

### 阶段三：运营计费

目标：算得清、卖得出去。

建设内容：客户门户、产品目录、订单、合同、卡时计费、账单、经营分析。

### 阶段四：AI生产与MaaS

目标：从算力交付升级为AI服务交付。

建设内容：Notebook、训练平台、模型仓库、推理服务、Token计量、模型API运营。

### 阶段五：智能运维

目标：降低故障率和运维成本。

建设内容：跨层根因分析、自动化Runbook、故障预测、资源推荐、智能助手。
