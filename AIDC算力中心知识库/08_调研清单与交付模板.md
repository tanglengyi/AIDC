# AIDC项目调研清单与交付模板

## 1. 文档用途

本文用于AIDC算力中心项目的售前调研、需求访谈、方案编写、范围界定和交付物组织。

---

## 2. 调研总原则

先判断平台边界，再拆功能模块；先确认业务闭环，再确认系统集成；先问验收指标，再讨论技术实现。

不要一上来问“要哪些功能”，而要先问：

- 谁建设
- 谁使用
- 谁付费
- 谁运维
- 卖什么资源
- 如何开通
- 如何计量
- 如何验收

---

## 9. 一期范围确认模板

### 9.1 项目定位

本期项目定位为：

- [ ] DCIM/设施管理平台
- [ ] GPU与IT资源纳管平台
- [ ] 算力资源调度平台
- [ ] AIDC综合运营平台
- [ ] AI开发平台
- [ ] MaaS模型服务平台

### 9.2 一期必须闭环

本期必须完成的业务闭环：

- [ ] 资产入库到资源池
- [ ] 客户申请到资源开通
- [ ] 任务提交到结果产出
- [ ] 用量采集到账单生成
- [ ] 告警发现到工单关闭
- [ ] 容量分析到扩容建议

### 9.3 明确不做

本期不做或只预留接口：

- [ ] 完整MLOps
- [ ] 完整MaaS
- [ ] 自动化高风险修复
- [ ] 替换现有DCIM
- [ ] 替换现有BSS/财务系统
- [ ] 跨地域统一调度
- [ ] 业务代码级APM

## 11. 验收指标模板

| 类别 | 指标 | 验收口径 |
|---|---|---|
| 资产 | 资产纳管率 | GPU服务器、机柜、U位、BMC纳管比例 |
| 监控 | 告警覆盖率 | 设施、GPU、网络、存储、调度系统覆盖情况 |
| 调度 | GPU利用率 | 资源池维度、租户维度、任务维度 |
| 交付 | 资源开通时长 | 从审批通过到资源可用 |
| 计量 | 用量准确率 | 与调度系统、日志、账单交叉校验 |
| 运维 | MTTR | 典型故障平均恢复时间 |
| 运营 | 账单生成周期 | 日账单/月账单生成和对账时间 |
| 管理 | 报表完整性 | SLA、收入、成本、容量、利用率 |
